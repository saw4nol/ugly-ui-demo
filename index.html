The Ugliest UI I Could Design

A deliberately bad UI and notes on how to improve it.

Screenshot of the "ugly" UI

What’s wrong with this UI?

  • Horrible color contrast and clashing hues.
  • Poor layout with misaligned elements and no spacing system.
  • Unreadable fonts (too many font sizes and styles used randomly).
  • Non-semantic HTML — accessibility ignored (no ARIA, no landmarks).
  • Confusing interaction patterns and unclear affordances.

What I would do to improve it

  1. Apply a consistent design system: typography scale, spacing, color palette, and elevation rules.
  2. Fix color contrast to meet WCAG AA at minimum — test with color-contrast tools.
  3. Use semantic HTML and ARIA attributes; ensure tab order and keyboard navigation work.
  4. Simplify the layout using a responsive grid or flexbox — mobile-first approach.
  5. Introduce consistent affordances and clear call-to-action buttons with clear language.
  6. Add unit / visual regression tests and user testing to validate improvements.
